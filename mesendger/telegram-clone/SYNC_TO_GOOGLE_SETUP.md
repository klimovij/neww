# Настройка автоматической синхронизации данных мониторинга времени с Google сервером

## 📋 Описание

Система автоматически отправляет данные о входе/выходе пользователей на Google сервер при записи в локальную базу данных. Это происходит прозрачно - при каждом вызове функции `addWorkTimeLog` данные автоматически дублируются на Google сервер.

## 🔧 Настройка на локальном ПК (где работает приложение с пользователями)

### 1. Установка переменных окружения

Добавьте в файл `.env` или установите переменные окружения системы:

```bash
# URL Google сервера (с /api в конце)
GOOGLE_SERVER_URL=https://your-google-server.com/api

# API ключ для аутентификации (должен совпадать с REMOTE_WORKTIME_API_KEY на Google сервере)
REMOTE_WORKTIME_API_KEY=your-secret-api-key-here
```

### 2. Пример настройки для разных окружений

**Windows (PowerShell):**
```powershell
$env:GOOGLE_SERVER_URL = "https://your-google-server.com/api"
$env:REMOTE_WORKTIME_API_KEY = "your-secret-api-key-here"
```

**Windows (CMD):**
```cmd
set GOOGLE_SERVER_URL=https://your-google-server.com/api
set REMOTE_WORKTIME_API_KEY=your-secret-api-key-here
```

**Linux/macOS:**
```bash
export GOOGLE_SERVER_URL="https://your-google-server.com/api"
export REMOTE_WORKTIME_API_KEY="your-secret-api-key-here"
```

### 3. Перезапуск сервера

После установки переменных окружения перезапустите Node.js сервер:

```bash
npm restart
# или
pm2 restart all
```

## 🔧 Настройка на Google сервере

### 1. Установка переменных окружения

На Google сервере установите:

```bash
# API ключ для аутентификации (должен совпадать с REMOTE_WORKTIME_API_KEY на локальном ПК)
REMOTE_WORKTIME_API_KEY=your-secret-api-key-here

# Флаг, что это Google сервер (чтобы данные не отправлялись обратно)
IS_GOOGLE_SERVER=true
```

**⚠️ ВАЖНО:** На Google сервере НЕ устанавливайте `GOOGLE_SERVER_URL`, чтобы избежать бесконечного цикла отправки данных.

### 2. Проверка работы

После настройки данные будут автоматически синхронизироваться:

1. **На локальном ПК:** При записи данных в локальную базу они автоматически отправляются на Google сервер
2. **На Google сервере:** Данные сохраняются в базу, но НЕ отправляются обратно (благодаря `IS_GOOGLE_SERVER=true`)

## 📊 Как это работает

1. На локальном ПК собираются данные из журнала Windows (события 4624, 4634)
2. Данные записываются в локальную базу через функцию `addWorkTimeLog()`
3. Функция автоматически отправляет данные на Google сервер (если настроены переменные окружения)
4. На Google сервере данные сохраняются в базу через API `/api/remote-worktime`
5. Данные доступны в модалке "Мониторинг времени" на Google сервере

## 🔍 Проверка работы

### На локальном ПК

Проверьте логи сервера - при записи данных должны появляться сообщения:

```
✅ Данные отправлены на Google сервер: Иван Иванов - login - 2025-01-20 08:30:15
```

Если отправка не удалась, вы увидите:

```
⚠️  Ошибка отправки данных на Google сервер: Connection timeout
```

### На Google сервере

1. Откройте модалку "Мониторинг времени" в приложении
2. Выберите период (например, вчерашний день)
3. Нажмите "Показать отчет"
4. Проверьте, что данные отображаются корректно

## ⚠️ Важные замечания

1. **API ключ должен совпадать** на локальном ПК и Google сервере
2. **Отправка не блокирует** запись в локальную базу - если отправка не удалась, данные все равно сохраняются локально
3. **Таймаут отправки:** 5 секунд - если Google сервер недоступен, операция завершится с ошибкой, но не прервет запись в базу
4. **Автоматическая защита от циклов:** Данные с Google сервера не отправляются обратно благодаря параметру `skipGoogleSync=true` и флагу `IS_GOOGLE_SERVER=true`

## 🐛 Устранение проблем

### Ошибка: "Connection timeout"

**Причина:** Google сервер недоступен или не отвечает

**Решение:**
- Проверьте доступность сервера: `ping your-google-server.com`
- Проверьте правильность URL в `GOOGLE_SERVER_URL`
- Проверьте настройки файрвола

### Данные не появляются на Google сервере

**Причина:** Неправильный API ключ или URL

**Решение:**
- Проверьте, что `REMOTE_WORKTIME_API_KEY` совпадает на обоих серверах
- Проверьте логи на Google сервере на наличие ошибок
- Проверьте, что маршрут `/api/remote-worktime` подключен в `server.js`

### Данные отправляются дважды

**Причина:** Цикл отправки данных

**Решение:**
- Убедитесь, что на Google сервере установлена переменная `IS_GOOGLE_SERVER=true`
- Убедитесь, что на Google сервере НЕ установлена переменная `GOOGLE_SERVER_URL`

## 📝 Примеры использования

### Полная настройка на локальном ПК

1. Создайте файл `.env` в корне проекта:
```
GOOGLE_SERVER_URL=https://your-google-server.com/api
REMOTE_WORKTIME_API_KEY=your-secret-api-key-here
```

2. Загрузите переменные окружения в Node.js (используя пакет `dotenv`):
```javascript
require('dotenv').config();
```

3. Перезапустите сервер

### Полная настройка на Google сервере

1. Создайте файл `.env`:
```
REMOTE_WORKTIME_API_KEY=your-secret-api-key-here
IS_GOOGLE_SERVER=true
```

2. Перезапустите сервер

## 🔄 Отключение синхронизации

Если нужно временно отключить отправку данных на Google сервер:

1. Удалите или закомментируйте переменные окружения:
```bash
# GOOGLE_SERVER_URL=https://your-google-server.com/api
# REMOTE_WORKTIME_API_KEY=your-secret-api-key-here
```

2. Перезапустите сервер

Данные будут продолжать сохраняться в локальную базу, но не будут отправляться на Google сервер.
