# Инструкция по установке системы мониторинга активности на ПК сотрудников

## Шаг 1: Подготовка файлов

1. Скопируйте папку `export_to_database` на ПК сотрудника (например, в `C:\Users\ИмяПользователя\web\pc-worktime`)

2. Откройте файл `pc_activity_agent.ps1` в блокноте и измените:
   ```powershell
   $UserUsername = "Имя_Фамилия"  # ← Замените на логин сотрудника (например: ivan_ivanov)
   ```

## Шаг 2: Настройка планировщика задач (автозапуск)

1. **Запустите PowerShell от имени администратора**

2. Перейдите в папку со скриптами:
   ```powershell
   cd "C:\Users\ИмяПользователя\web\pc-worktime"
   ```

3. Запустите скрипт настройки:
   ```powershell
   .\setup_pc_worktime_tasks.ps1
   ```

4. Скрипт автоматически создаст 3 задачи:
   - `PC_Worktime_On` - отправка события включения ПК
   - `PC_Worktime_Off` - отправка события выключения ПК
   - `PC_Activity_Agent` - отслеживание активности (автозапуск при входе в систему)

## Шаг 3: Проверка работы

1. Откройте планировщик задач Windows (Task Scheduler)

2. Найдите задачу `PC_Activity_Agent` и проверьте, что она **включена** (State = Ready)

3. Запустите задачу вручную (правой кнопкой → Выполнить) и проверьте:
   - В консоли PowerShell должно появиться сообщение "Starting activity agent"
   - Данные должны собираться каждую минуту

4. Проверьте логи в папке `C:\pc-worktime\logs\activity_YYYY-MM-DD.jsonl`

## Шаг 4: Исключение в антивирусе (важно!)

Если антивирус блокирует скрипт:

1. Откройте **Windows Defender** (Защита от вирусов и угроз)

2. Перейдите в **Управление настройками** → **Исключения**

3. Добавьте исключение для **папки**: `C:\Users\ИмяПользователя\web\pc-worktime`

## Шаг 5: Настройка на сервере

На сервере должен быть создан пользователь с логином, совпадающим с `$UserUsername` в скрипте:

1. Зайдите на сервер (http://35.232.108.72)

2. В админ-панели создайте/обновите пользователя:
   - **Username**: `ivan_ivanov` (латиница, без пробелов)
   - **ФИО**: `Иван Иванов` (русские буквы)

3. Теперь в модалке "Мониторинг времени" будет отображаться правильное ФИО

## Что отслеживается:

- ✅ Время включения/выключения ПК
- ✅ Активность пользователя (активные/простаивающие минуты)
- ✅ Приложения, которые используются
- ✅ Сайты, открытые в браузере (Chrome, Edge, Firefox)
- ✅ Скриншоты экрана (каждые 15 минут)

## Где смотреть данные:

1. В веб-интерфейсе: **Мониторинг времени** → выберите дату → нажмите **"Детали"**

2. В модалке "Детали" есть 3 вкладки:
   - **События** - входы/выходы
   - **Сайты** - открытые URL
   - **Скриншоты** - снимки экрана

## Устранение проблем:

### Скрипт не запускается
- Проверьте, что PowerShell разрешает выполнение скриптов: `Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser`

### Данные не отправляются на сервер
- Проверьте подключение к интернету
- Проверьте логи в `C:\pc-worktime\logs\activity_errors.log`
- Проверьте, что API ключ правильный: `BsKFpZmdp6ocPKUD6g6YxTgMSTZEaPZXkbddxsifERA=`

### В модалке отображается логин вместо ФИО
- Проверьте, что на сервере в таблице `users` поле `username` совпадает с `$UserUsername` в скрипте
- Проверьте, что поле `fio` заполнено на сервере

## Контакты для поддержки:

При возникновении проблем проверьте:
1. Логи скрипта в консоли PowerShell
2. Логи на сервере: `pm2 logs mesendger-server`
3. Логи ошибок: `C:\pc-worktime\logs\activity_errors.log`
