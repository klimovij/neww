# Установка задачи отслеживания выключения ПК

## Описание

Система отслеживания выключения ПК с резервным сохранением данных. Если данные не удается отправить сразу при выключении, они сохраняются в файл и отправляются при следующем запуске ПК.

## Компоненты

1. **send_pc_shutdown_with_fallback.ps1** - Скрипт отправки данных о выключении с резервным сохранением
2. **setup_shutdown_task.ps1** - Скрипт автоматической установки задачи в планировщике задач Windows
3. **send_pc_startup.ps1** - Модифицированный скрипт запуска, который проверяет и отправляет сохраненные данные о выключении

## Как это работает

### Вариант 1: Отправка при выключении (основной)
1. При выключении ПК планировщик задач запускает `send_pc_shutdown_with_fallback.ps1`
2. Скрипт пытается отправить данные о выключении на сервер
3. Если отправка успешна - данные отправлены, файл с отложенными данными удаляется (если был)

### Вариант 2: Резервное сохранение (fallback)
1. Если отправка не удалась (нет интернета, сервер недоступен, ПК выключается слишком быстро):
   - Данные сохраняются в файл `%APPDATA%\mesendger\pending_shutdown.json`
   - Скрипт завершается с кодом успеха (чтобы не блокировать выключение)

### Вариант 3: Отправка при следующем запуске
1. При запуске ПК выполняется `send_pc_startup.ps1`
2. Скрипт проверяет наличие файла `pending_shutdown.json`
3. Если файл найден:
   - Читает данные о выключении
   - Отправляет их на сервер вместе с данными о включении
   - Удаляет файл после успешной отправки

## Установка

### Автоматическая установка (рекомендуется)

1. Запустите PowerShell от имени администратора
2. Перейдите в папку со скриптами:
   ```powershell
   cd "C:\путь\к\папке\export_to_database"
   ```
3. Запустите скрипт установки:
   ```powershell
   .\setup_shutdown_task.ps1 [username]
   ```
   Где `[username]` - опциональный параметр (username сотрудника)

4. Скрипт:
   - Скопирует `send_pc_shutdown_with_fallback.ps1` в `C:\pc-worktime`
   - Создаст задачу в планировщике задач Windows
   - Настроит запуск при событии выключения системы

### Ручная установка

1. Скопируйте `send_pc_shutdown_with_fallback.ps1` в `C:\pc-worktime`
2. Создайте задачу в планировщике задач:
   - Триггер: Событие выключения системы (Event ID 1074, 1076, 6008)
   - Действие: Запуск PowerShell скрипта
   - Команда: `powershell.exe -ExecutionPolicy Bypass -WindowStyle Hidden -File "C:\pc-worktime\send_pc_shutdown_with_fallback.ps1" "username"`

## Проверка работы

### Проверка задачи в планировщике
```powershell
schtasks /Query /TN Mesendger_PC_Shutdown_With_Fallback
```

### Ручной запуск скрипта (для тестирования)
```powershell
.\send_pc_shutdown_with_fallback.ps1 "Ksendzik_Oleg"
```

### Проверка логов
- Лог выключения: `%APPDATA%\mesendger\pc_shutdown.log`
- Лог запуска: `%APPDATA%\mesendger\pc_startup.log`
- Отложенные данные: `%APPDATA%\mesendger\pending_shutdown.json`

## Варианты реализации

### Текущая реализация (рекомендуется)
- ✅ Использует планировщик задач Windows с событием выключения
- ✅ Резервное сохранение в файл при ошибке
- ✅ Автоматическая отправка при следующем запуске
- ✅ Надежно работает даже при быстром выключении

### Альтернативные варианты

#### Вариант A: WMI подписка (более сложный)
- Использует `Register-WmiEvent` для отслеживания событий выключения
- Требует постоянно работающий процесс
- Может быть менее надежным при быстром выключении

#### Вариант B: Системный хук (требует C# код)
- Использует Windows API для перехвата событий выключения
- Более сложная реализация
- Требует компиляции

#### Вариант C: Периодическая проверка (менее эффективный)
- Проверяет системные события каждые N секунд
- Может пропустить событие выключения
- Потребляет ресурсы постоянно

## Требования

- Windows 7/8/10/11
- PowerShell 5.1 или выше
- Права администратора для установки задачи
- Доступ к интернету для отправки данных на сервер

## Устранение неполадок

### Задача не запускается при выключении
1. Проверьте, что задача создана: `schtasks /Query /TN Mesendger_PC_Shutdown_With_Fallback`
2. Проверьте права доступа к скрипту
3. Проверьте логи планировщика задач Windows

### Данные не отправляются
1. Проверьте логи: `%APPDATA%\mesendger\pc_shutdown.log`
2. Проверьте наличие файла с отложенными данными: `%APPDATA%\mesendger\pending_shutdown.json`
3. Проверьте доступность сервера: `http://35.232.108.72/api/worktime`

### Данные не отправляются при запуске
1. Проверьте, что `send_pc_startup.ps1` модифицирован (добавлена функция `Send-PendingShutdown`)
2. Проверьте логи запуска: `%APPDATA%\mesendger\pc_startup.log`
3. Убедитесь, что файл `pending_shutdown.json` существует

## Примечания

- Данные о выключении отображаются в модалке "Подробности" вместе с данными о входе/выходе
- Время выключения сохраняется точно, даже если отправка произошла при следующем запуске
- Система автоматически обрабатывает случаи, когда интернет недоступен при выключении
