# 📊 УСТАНОВКА ЭКСПОРТА ДАННЫХ РАБОЧЕГО ВРЕМЕНИ

## 🎯 Назначение

Автоматическая отправка данных о входах/выходах сотрудников с удаленного сервера на Google сервер.

**Что делает:**
- ✅ Каждое утро в **7:00** запускается автоматически
- ✅ Читает данные за **вчерашний день** из локальной базы `messenger.db`
- ✅ Отправляет данные на Google сервер через API
- ✅ Данные отображаются в модалке **"Отчет Удаленка"**

---

## 📋 Требования

На удаленном сервере должны быть установлены:
- ✅ Windows (любая версия)
- ✅ PowerShell 5.0+
- ✅ Node.js (https://nodejs.org/)
- ✅ Приложение Mesendger (с базой данных `messenger.db`)

---

## 🚀 УСТАНОВКА (НА УДАЛЕННОМ СЕРВЕРЕ)

### Шаг 1: Скопируйте файлы

Скопируйте эти файлы на удаленный сервер в папку `export_to_database`:

```
export_to_database/
├── export_remote_worktime.ps1         ← основной скрипт
├── setup_remote_worktime_export.ps1   ← установка задачи
└── Mesendger_Remote_Worktime_Export.xml ← конфигурация задачи
```

Например, если приложение находится по пути:
```
C:\Users\Admin\mesendger\telegram-clone\
```

То файлы должны быть здесь:
```
C:\Users\Admin\mesendger\telegram-clone\export_to_database\
```

---

### Шаг 2: Откройте PowerShell **от имени Администратора**

1. Нажмите **Win + X**
2. Выберите **"Windows PowerShell (Администратор)"**
3. Перейдите в папку:

```powershell
cd "C:\Users\Admin\mesendger\telegram-clone\export_to_database"
```

---

### Шаг 3: Разрешите выполнение скриптов

```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser -Force
```

---

### Шаг 4: Запустите установку

```powershell
.\setup_remote_worktime_export.ps1
```

**Что произойдет:**
1. ✅ Проверится наличие Node.js
2. ✅ Проверится наличие базы данных
3. ✅ Установится модуль sqlite3 (если нужно)
4. ✅ Создастся задача в планировщике
5. ✅ Задача настроится на запуск каждый день в 7:00

## ⚠️ РЕШЕНИЕ ПРОБЛЕМ

### Проблема 1: "Database file not found"

**Причина:** База данных messenger.db не найдена

**Решение:**
1. Убедитесь, что приложение Mesendger запущено и создало базу
2. Проверьте путь к базе:
```powershell
# Проверьте оба пути:
Test-Path "путь\telegram-clone\server\messenger.db"
Test-Path "путь\server\messenger.db"
```

---

### Проблема 2: "Node.js not found"

**Причина:** Node.js не установлен или не добавлен в PATH

**Решение:**
1. Скачайте Node.js: https://nodejs.org/
2. Установите (выберите "Add to PATH")
3. Перезапустите PowerShell
4. Проверьте: `node --version`

---

### Проблема 3: "No events found for date"

**Причина:** В базе нет данных за указанную дату

**Решение:**
1. Проверьте, работает ли система логирования на удаленном ПК
2. Проверьте базу данных вручную:
```powershell
# Запустите SQLite
cd "путь\telegram-clone\server"
npm install -g sqlite3
sqlite3 messenger.db

# Внутри SQLite:
SELECT COUNT(*) FROM work_time_logs WHERE date(event_time) = '2025-11-30';
SELECT DISTINCT date(event_time) FROM work_time_logs ORDER BY event_time DESC LIMIT 10;
.exit
```

---

### Проблема 4: "Invalid API key"

**Причина:** API ключ не совпадает с ключом на Google сервере

**Решение:**
1. Проверьте файл `.env` в папке `server`:
```
REMOTE_WORKTIME_API_KEY=BsKFpZmdp6ocPKUD6g6YxTgMSTZEaPZXkbddxsifERA=
```
2. Убедитесь, что ключ совпадает с ключом на Google сервере

---

### Проблема 5: Задача не запускается автоматически

**Причина:** Неправильные права или настройки задачи

**Решение:**
1. Откройте **Планировщик задач** (`taskschd.msc`)
2. Найдите задачу: **Mesendger_Remote_Worktime_Export**
3. Правая кнопка → **"Свойства"**
4. Вкладка **"Общие"**:
   - ✅ **"Выполнять с наивысшими правами"** - включено
   - ✅ **"Выполнять для пользователя вне зависимости от входа"** - НЕ включено
5. Вкладка **"Триггеры"**:
   - ✅ Время: **07:00:00**
   - ✅ Повторять: **Ежедневно**
   - ✅ Включено: **Да**
6. Вкладка **"Действия"**:
   - ✅ Программа: `powershell.exe`
   - ✅ Аргументы: `-NoProfile -ExecutionPolicy Bypass -WindowStyle Hidden -File "путь\export_remote_worktime.ps1"`

---

### Проблема 6: Данные приходят не всегда

**Возможные причины и решения:**

#### A) База данных заблокирована
```powershell
# Убедитесь, что приложение не блокирует базу
# Скрипт открывает базу в режиме READONLY, поэтому конфликтов быть не должно
```

#### B) Ошибка в SQL запросе
```powershell
# Скрипт поддерживает разные форматы дат:
# - YYYY-MM-DD
# - DD.MM.YYYY HH:MM:SS
# - ISO формат с временной зоной
```

#### C) Нет сетевого подключения
```powershell
# Проверьте доступ к Google серверу:
Test-NetConnection -ComputerName 35.232.108.72 -Port 5000
```

#### D) Timeout при отправке
```powershell
# Скрипт использует timeout 30 секунд
# Если данных много (>1000 записей), может не хватить времени
# Решение: увеличьте timeout в скрипте или отправляйте батчами
```

## 🔧 НАСТРОЙКА ВРЕМЕНИ ЗАПУСКА

Если хотите изменить время запуска (например, на 8:00):

1. Откройте файл `Mesendger_Remote_Worktime_Export.xml`
2. Найдите строку:
```xml
<StartBoundary>2025-12-01T07:00:00</StartBoundary>
```
3. Измените на нужное время:
```xml
<StartBoundary>2025-12-01T08:00:00</StartBoundary>
```
4. Переустановите задачу:
```powershell
.\setup_remote_worktime_export.ps1
```

## 💡 ДОПОЛНИТЕЛЬНО

### Отправка данных за несколько дней

Если нужно отправить данные за несколько дней (например, после простоя):

```powershell
# За последние 7 дней
for ($i = 1; $i -le 7; $i++) {
    $date = (Get-Date).AddDays(-$i).ToString("yyyy-MM-dd")
    Write-Host "Отправка данных за $date" -ForegroundColor Cyan
    .\export_remote_worktime.ps1 -Date $date
    Start-Sleep -Seconds 2
}
```

### Отправка данных за конкретный месяц

```powershell
# За ноябрь 2025
1..30 | ForEach-Object {
    $date = "2025-11-$(${_}.ToString('00'))"
    Write-Host "Отправка данных за $date" -ForegroundColor Cyan
    .\export_remote_worktime.ps1 -Date $date
    Start-Sleep -Seconds 1
}
```
