# Быстрое исправление автозапуска агента активности

## Проблема
Задача `PC_Activity_Agent` существует, но не запускается автоматически при перезагрузке ПК.

## Решение (займет 2 минуты)

### Шаг 1: Откройте PowerShell от имени администратора
1. Нажмите `Win + X`
2. Выберите **Windows PowerShell (администратор)** или **Терминал (администратор)**

### Шаг 2: Перейдите в папку со скриптами
```powershell
cd "C:\Users\Ronin\web\pc-worktime"
```

### Шаг 3: Запустите скрипт установки
```powershell
.\setup_pc_worktime_tasks.ps1
```

Скрипт:
- ✅ Удалит старую задачу (если она настроена неправильно)
- ✅ Создаст новую задачу с правильными настройками
- ✅ Настроит автозапуск при входе в систему
- ✅ Добавит перезапуск при сбое

### Шаг 4: Проверьте результат
После выполнения скрипта вы должны увидеть:
```
✅ Создана/обновлена задача: PC_Activity_Agent
✅ Задача включена и готова к запуску
```

### Шаг 5: Проверьте задачу в планировщике
1. Откройте **Планировщик задач** (`Win + R` → `taskschd.msc`)
2. Найдите задачу `PC_Activity_Agent`
3. Проверьте:
   - ✅ Статус: **Готово** (Ready)
   - ✅ Вкладка **Триггеры**: должен быть включен триггер "При входе в систему"
   - ✅ Вкладка **Действия**: должен быть указан правильный путь к скрипту

### Шаг 6: Перезагрузите ПК
1. Перезагрузите компьютер
2. После входа в систему подождите 1-2 минуты
3. Откройте **Диспетчер задач** (`Ctrl + Shift + Esc`)
4. Вкладка **Процессы** → найдите `powershell.exe`
5. Проверьте папку `C:\pc-worktime\logs\` — должны появиться новые файлы логов

## Если проблема сохраняется

### Вариант 1: Запустить задачу вручную
1. Откройте **Планировщик задач**
2. Найдите задачу `PC_Activity_Agent`
3. Правой кнопкой → **Выполнить** (Run)
4. Проверьте логи в папке `C:\pc-worktime\logs\`

### Вариант 2: Проверить путь к скрипту
Убедитесь, что скрипт находится по пути:
```
C:\Users\Ronin\web\pc-worktime\pc_activity_agent.ps1
```

Если скрипт находится в другом месте, измените путь в задаче планировщика или переместите скрипт в правильную папку.

### Вариант 3: Проверить права доступа
1. Правой кнопкой на скрипт `pc_activity_agent.ps1`
2. **Свойства** → вкладка **Безопасность**
3. Убедитесь, что ваш пользователь имеет права на **Чтение и выполнение**

### Вариант 4: Проверить антивирус
Если антивирус блокирует скрипт:
1. Откройте **Защитник Windows**
2. **Защита от вирусов и угроз** → **Управление настройками**
3. **Исключения** → **Добавить или удалить исключения**
4. Добавьте исключение для папки: `C:\Users\Ronin\web\pc-worktime`

## Проверка работы агента

### Через 5-10 минут после запуска проверьте:
1. **Логи агента**: `C:\pc-worktime\logs\agent_YYYY-MM-DD.log`
2. **Логи активности**: `C:\pc-worktime\logs\activity_YYYY-MM-DD.jsonl`
3. **Скриншоты**: `C:\pc-worktime\screenshots\`

### В логах должны быть строки:
```
✅ Successfully sent batch: imported=X, total=X
✅ Screenshot sent successfully: True
```

### В веб-интерфейсе:
Откройте веб-интерфейс мониторинга и проверьте:
- Появляется ли активность сотрудника
- Отображаются ли скриншоты
- Отображаются ли активные сайты (URLs)
