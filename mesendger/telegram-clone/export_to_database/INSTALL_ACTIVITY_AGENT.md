# Инструкция по установке агента активности на ПК сотрудника

## Быстрая установка (автоматическая)

1. **Скопируйте файлы на ПК сотрудника:**
   - `pc_activity_agent.ps1` → `C:\Users\<ИмяПользователя>\web\pc-worktime\pc_activity_agent.ps1`
   - `setup_pc_worktime_tasks.ps1` → в ту же папку

2. **Откройте PowerShell от имени администратора** и выполните:
   ```powershell
   cd "C:\Users\<ИмяПользователя>\web\pc-worktime"
   .\setup_pc_worktime_tasks.ps1
   ```

3. **В скрипте `pc_activity_agent.ps1` измените строку 39:**
   ```powershell
   $UserUsername = "Ksendzik_Oleg"  # ← замените на логин сотрудника (например: ivan_ivanov, petrov_petr)
   ```

4. **Перезагрузите ПК** для проверки автозапуска.

## Ручная установка (если автоматическая не работает)

### Шаг 1: Настройка скрипта агента

1. Откройте файл `pc_activity_agent.ps1` в Блокноте
2. Найдите строку 39 и измените логин сотрудника:
   ```powershell
   $UserUsername = "Логин_Сотрудника"  # Например: ivan_ivanov, petrov_petr
   ```
3. Сохраните файл

### Шаг 2: Создание задачи в планировщике Windows

1. Откройте **Планировщик задач** (Task Scheduler)
2. Нажмите **Создать задачу** (Create Task) справа
3. Вкладка **Общие** (General):
   - Имя: `PC_Activity_Agent`
   - Описание: `Отслеживание активности пользователя на ПК`
   - Выполнять для всех пользователей: ✅
   - Выполнять с наивысшими правами: ✅
   - Настроить для: **Windows 10** (или ваша версия Windows)

4. Вкладка **Триггеры** (Triggers) → **Создать** (New):
   - Начать задачу: **При входе в систему**
   - Пользователь: выберите пользователя ПК
   - Задержка: **30 секунд**
   - ✅ Включено
   - Нажмите **OK**

5. Вкладка **Действия** (Actions) → **Создать** (New):
   - Действие: **Запуск программы**
   - Программа или сценарий: `powershell.exe`
   - Добавить аргументы:
     ```
     -ExecutionPolicy Bypass -WindowStyle Hidden -File "C:\Users\<ИмяПользователя>\web\pc-worktime\pc_activity_agent.ps1" -User <ИмяПользователя>
     ```
     (Замените `<ИмяПользователя>` на имя пользователя ПК, например: `Ronin`)
   - Начало в: `C:\Users\<ИмяПользователя>\web\pc-worktime`
   - Нажмите **OK**

6. Вкладка **Условия** (Conditions):
   - ✅ Запускать задачу только при наличии сетевого подключения
   - ✅ Пробуждать компьютер для выполнения задачи
   - ✅ Запускать при питании от электросети
   - ✅ Запускать при питании от батареи (для ноутбуков)

7. Вкладка **Параметры** (Settings):
   - ✅ Разрешить выполнение задачи по требованию
   - ✅ Выполнять задачу немедленно, если пропущен плановый запуск
   - ✅ Если задача не удалось запустить, запустить повторно через: **2 минуты**
   - Количество попыток повторного запуска: **5**
   - ✅ Останавливать задачу, выполняющуюся дольше: **НЕТ** (снять галочку)
   - Если запущенная задача не завершается при запросе: **Остановить существующий экземпляр**
   - Нажмите **OK**

8. Сохраните задачу. Введите пароль администратора, если потребуется.

### Шаг 3: Проверка установки

1. Откройте **Планировщик задач**
2. Найдите задачу `PC_Activity_Agent`
3. Проверьте статус задачи — должно быть **Готово** (Ready)
4. Правой кнопкой → **Выполнить** (Run) для проверки запуска
5. Проверьте логи в папке `C:\pc-worktime\logs\` — должны появиться новые файлы

### Шаг 4: Автозапуск при перезагрузке

1. Перезагрузите ПК
2. После входа в систему подождите 1-2 минуты
3. Откройте **Диспетчер задач** (Task Manager) → вкладка **Процессы**
4. Найдите процесс `powershell.exe` — должен быть запущен процесс `pc_activity_agent.ps1`
5. Или проверьте папку `C:\pc-worktime\logs\` — должны появиться новые файлы логов

## Проверка работы агента

### 1. Проверка логов
Откройте папку `C:\pc-worktime\logs\` и проверьте наличие файлов вида:
- `activity_YYYY-MM-DD.jsonl` — логи активности
- `agent_YYYY-MM-DD.log` — логи агента

### 2. Проверка отправки данных
Через 5-10 минут после запуска проверьте логи агента на наличие строк:
```
✅ Successfully sent batch: imported=X, total=X
✅ Screenshot sent successfully: True
```

### 3. Проверка на сервере
Откройте веб-интерфейс мониторинга и проверьте:
- Появляется ли активность сотрудника
- Отображаются ли скриншоты
- Отображаются ли активные сайты (URLs)

## Устранение проблем

### Проблема: Задача не запускается автоматически

**Решение:**
1. Откройте **Планировщик задач**
2. Найдите задачу `PC_Activity_Agent`
3. Проверьте статус — если **Отключено** (Disabled), включите задачу
4. Проверьте триггеры — должен быть включен триггер "При входе в систему"
5. Проверьте права доступа — задача должна выполняться от имени пользователя

### Проблема: PowerShell скрипт блокируется антивирусом

**Решение:**
1. Откройте **Защитник Windows** (Windows Security)
2. Перейдите в **Защита от вирусов и угроз** → **Управление настройками**
3. Прокрутите вниз до **Исключения** → **Добавить или удалить исключения**
4. Добавьте исключения:
   - **Файл**: `C:\Users\<ИмяПользователя>\web\pc-worktime\pc_activity_agent.ps1`
   - **Папка**: `C:\Users\<ИмяПользователя>\web\pc-worktime`
   - **Папка**: `C:\pc-worktime`

### Проблема: Ошибка "ExecutionPolicy"

**Решение:**
Откройте PowerShell от имени администратора и выполните:
```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser -Force
```

### Проблема: Скрипт не отправляет данные на сервер

**Решение:**
1. Проверьте подключение к интернету
2. Проверьте доступность сервера:
   ```powershell
   Test-NetConnection -ComputerName 35.232.108.72 -Port 80
   ```
3. Проверьте логи агента в папке `C:\pc-worktime\logs\` на наличие ошибок
4. Убедитесь, что в скрипте `pc_activity_agent.ps1` правильно указан `$UserUsername` (строка 39)

## Настройка для всех сотрудников (40 человек)

1. Создайте список сотрудников с их логинами (соответствие `username` в базе данных)
2. Для каждого ПК:
   - Скопируйте файлы агента
   - Измените `$UserUsername` в скрипте
   - Запустите `setup_pc_worktime_tasks.ps1` от имени администратора
   - Или создайте задачу вручную по инструкции выше
3. Проверьте работу через веб-интерфейс мониторинга

## Важные замечания

- ✅ Скрипт агента работает в скрытом режиме (`-WindowStyle Hidden`)
- ✅ Агент автоматически перезапускается при сбое (до 5 попыток)
- ✅ Данные отправляются на сервер каждые 5 минут
- ✅ Скриншоты делаются каждые 15 минут
- ✅ Логи сохраняются локально в папке `C:\pc-worktime\logs\`
- ⚠️ Агент должен работать постоянно для корректного мониторинга
- ⚠️ При изменении логина пользователя на сервере обновите `$UserUsername` в скрипте
