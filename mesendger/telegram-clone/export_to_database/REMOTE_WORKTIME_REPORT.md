# Отчет Удаленка - Инструкция

## Описание

Функционал "Отчет Удаленка" позволяет просматривать отчеты о входе/выходе пользователей с удаленных компьютеров за выбранную дату.

## Как использовать

### 1. Отправка данных с удаленного ПК

На удаленном компьютере (где работает локальное приложение) необходимо настроить отправку данных на сервер:

#### Автоматическая отправка за вчерашний день:

Используйте скрипт `send_yesterday_to_google.ps1`:
```powershell
cd "C:\path\to\mesendger\telegram-clone\export_to_database"
.\send_yesterday_to_google.ps1
```

#### Отправка за конкретную дату:

Используйте скрипт `send_date_to_google_simple.ps1`:
```powershell
cd "C:\path\to\mesendger\telegram-clone\export_to_database"
.\send_date_to_google_simple.ps1 -Date "2025-01-15"
```

#### Настройка планировщика задач (автоматическая отправка):

Можно настроить автоматическую отправку данных каждый день:

1. Откройте "Планировщик задач" (Task Scheduler)
2. Создайте новую задачу
3. Настройте триггер: ежедневно в 00:00 или позже
4. Действие: запустить PowerShell скрипт `send_yesterday_to_google.ps1`

### 2. Просмотр отчета в веб-интерфейсе

1. Откройте модалку "Мониторинг времени" в веб-приложении
2. Нажмите кнопку **"Отчет Удаленка"**
3. В открывшейся модалке вы увидите:
   - По умолчанию данные за вчерашний день
   - Список пользователей с первым входом и последним выходом
   - Вычисленное время работы
   - Кнопку "Просмотреть" для каждого пользователя

4. Для просмотра деталей пользователя:
   - Нажмите кнопку "Просмотреть" напротив нужного пользователя
   - В правой панели отобразятся все события входа/выхода за выбранный день

5. Для просмотра другого дня:
   - Измените дату в поле выбора даты в заголовке модалки
   - Отчет автоматически обновится

## API Endpoints

### Получение отчета за дату
```
GET /api/remote-worktime-report?date=YYYY-MM-DD
```

Если дата не указана, возвращаются данные за вчерашний день.

### Получение всех событий пользователя за дату
```
GET /api/remote-worktime-user-events?username=USERNAME&date=YYYY-MM-DD
```

## Формат данных

Данные отправляются в формате:
```json
{
  "events": [
    {
      "username": "user_name",
      "event_type": "login" | "logout",
      "event_time": "2025-01-15T09:00:00+00:00",
      "event_id": 4624
    }
  ]
}
```

## Примечания

- Скрипты отправки данных должны иметь доступ к локальной базе данных `messenger.db`
- Скрипты должны иметь доступ к серверу через API ключ
- API ключ находится в файле `.env` или используется по умолчанию: `BsKFpZmdp6ocPKUD6g6YxTgMSTZEaPZXkbddxsifERA=`
- URL сервера по умолчанию: `http://35.232.108.72`
