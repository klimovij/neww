# 🚀 Инструкция: Установка мониторинга на 15 ПК в офисе

## ✅ Скрипты полностью универсальные!

**НЕ НУЖНО ничего менять** - все пути, URL сервера и настройки одинаковые для всех ПК.  
**Единственное отличие** - username каждого сотрудника (запрашивается автоматически при установке).

---

## 📋 Что будет установлено на каждом ПК:

### Файлы:
```
C:\pc-worktime\
├── send_pc_startup.ps1              (отправка данных включения)
├── send_pc_shutdown_with_fallback.ps1  (отправка данных выключения)
├── send_pc_shutdown.ps1             (старая версия, для совместимости)
├── send_activity.ps1                (скриншоты, сайты, приложения)
├── monitor_shutdown.ps1             (фоновый мониторинг)
└── XML файлы задач (4 файла)

%APPDATA%\mesendger\
├── agent_config.json                (username и настройки)
└── Логи (создаются автоматически)
```

### Задачи в планировщике:
- ✅ **Mesendger_PC_Startup_Monitor** - включение ПК
- ✅ **Mesendger_PC_Shutdown_Monitor** - выключение ПК  
- ✅ **Mesendger_PC_Activity_Monitor** - скриншоты, сайты, приложения
- ⚠️ **Mesendger_PC_Logoff_Monitor** - дополнительный мониторинг выхода (опционально)

---

## 🎯 Вариант 1: Установка через флешку (рекомендуется)

### Шаг 1: Подготовьте флешку (один раз)

1. Скопируйте на флешку всю папку:
   ```
   C:\Users\Ronin\Desktop\mesendger-god\mesendger\telegram-clone\export_to_database\
   ```

2. На флешке будет папка `export_to_database` со всеми файлами

### Шаг 2: Установка на каждом ПК (3 минуты)

**На каждом из 15 компьютеров:**

1. **Вставьте флешку**

2. **Откройте PowerShell от имени администратора:**
   - Правой кнопкой на меню Пуск
   - "Windows PowerShell (администратор)" или "Терминал (Администратор)"

3. **Перейдите в папку на флешке:**
   ```powershell
   # Замените E:\ на букву вашей флешки
   cd E:\export_to_database
   ```

4. **Запустите установку:**
   ```powershell
   .\setup_pc_monitoring.ps1
   ```

5. **Скрипт спросит:**
   - **Username сотрудника** (например: `Ivanov_Ivan`, `Petrov_Petr`)  
     ⚠️ Должен совпадать с username в базе Mesendger!
   
   - **Интервал скриншотов** (по умолчанию 5 минут - просто нажмите Enter)
   
   - **Интервал сайтов** (по умолчанию 5 минут - просто нажмите Enter)
   
   - **Интервал приложений** (по умолчанию 5 минут - просто нажмите Enter)

6. **Готово!** Скрипт автоматически:
   - Создаст папку `C:\pc-worktime`
   - Скопирует все файлы
   - Обновит XML с username сотрудника
   - Создаст задачи в планировщике
   - Сохранит настройки

**Время установки: ~3 минуты на один ПК**

## 🎯 Вариант 2: Установка через сетевой диск (быстрее)

### Шаг 1: Поместите папку на сервер (один раз)

1. Скопируйте папку на сетевой диск:
   ```
   \\server\share\pc-monitoring\export_to_database\
   ```

2. Дайте права на чтение всем сотрудникам

### Шаг 2: Установка на каждом ПК

**На каждом из 15 компьютеров:**

1. **Откройте PowerShell от имени администратора**

2. **Запустите установку с сетевого диска:**
   ```powershell
   cd \\server\share\pc-monitoring\export_to_database
   .\setup_pc_monitoring.ps1
   ```

3. **Введите username сотрудника**

4. **Нажмите Enter для интервалов по умолчанию**

5. **Готово!**

## 🎯 Вариант 3: Массовая установка (для продвинутых)

Если у вас есть доступ к удаленному управлению ПК (например, через домен Active Directory):

### Создайте bat-файл для автозапуска:

```batch
@echo off
echo ========================================
echo Установка мониторинга Mesendger
echo ========================================

REM Проверка прав администратора
net session >nul 2>&1
if %errorlevel% neq 0 (
    echo ОШИБКА: Требуются права администратора!
    pause
    exit /b 1
)

REM Путь к установочной папке (измените на свой)
set SOURCE=\\server\share\pc-monitoring\export_to_database

REM Запуск установки с автоматическими ответами (требует доработки скрипта)
powershell -ExecutionPolicy Bypass -File "%SOURCE%\setup_pc_monitoring.ps1"

pause
```

## ✅ Проверка после установки

На каждом ПК проверьте:

### 1. Проверьте задачи:
```powershell
schtasks /Query /TN Mesendger_PC_Startup_Monitor
schtasks /Query /TN Mesendger_PC_Shutdown_Monitor
schtasks /Query /TN Mesendger_PC_Activity_Monitor
```

Должны показать: **"Состояние: Готово"** или **"Состояние: Выполняется"**

### 2. Проверьте файлы:
```powershell
dir C:\pc-worktime
dir $env:APPDATA\mesendger
```

### 3. Проверьте работу:
1. Перезагрузите ПК
2. Откройте модалку "Отчет активности локальных ПК" в Mesendger
3. Проверьте, что появились данные о включении

### 4. Проверьте логи (если нужно):
```powershell
notepad $env:APPDATA\mesendger\pc_startup.log
notepad $env:APPDATA\mesendger\pc_shutdown.log
notepad $env:APPDATA\mesendger\activity.log
```

## ⚠️ Важные моменты

### 1. Username сотрудника
- **Должен точно совпадать** с username в базе данных Mesendger
- **Формат:** обычно `Familiya_Imya` (например: `Ksendzik_Oleg`)
- Если не знаете username - проверьте в админке Mesendger

### 2. Права администратора
- Установка **ТРЕБУЕТ прав администратора**
- Убедитесь, что у вас есть admin-пароль для каждого ПК

### 3. Сеть
- Все ПК должны иметь доступ к серверу `http://35.232.108.72`
- Проверьте firewall, если данные не отправляются

### 4. Антивирус
- Некоторые антивирусы могут блокировать PowerShell скрипты
- Если установка не работает, добавьте папку `C:\pc-worktime` в исключения

## 🔧 Устранение проблем

### Проблема: "Скрипт не запускается"
**Решение:**
```powershell
Set-ExecutionPolicy RemoteSigned -Scope CurrentUser
```

### Проблема: "Задачи не создаются"
**Решение:**
- Убедитесь, что PowerShell запущен от имени администратора
- Проверьте, что XML файлы скопировались в `C:\pc-worktime\`

### Проблема: "Данные не отправляются"
**Решение:**
1. Проверьте логи: `%APPDATA%\mesendger\*.log`
2. Проверьте доступ к серверу: `Test-NetConnection 35.232.108.72 -Port 80`
3. Проверьте, что задачи запущены: `schtasks /Query /TN Mesendger_PC_Activity_Monitor`

### Проблема: "Неправильный username"
**Решение:**
1. Откройте файл: `%APPDATA%\mesendger\agent_config.json`
2. Исправьте username
3. Или запустите `setup_pc_monitoring.ps1` заново
