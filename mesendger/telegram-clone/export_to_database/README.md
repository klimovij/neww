# 📦 Мониторинг ПК для Mesendger

Система автоматической отправки данных о включении/выключении ПК, скриншотов, посещенных сайтов и приложений.

---

## 🚀 БЫСТРЫЙ СТАРТ

### 🏢 ДЛЯ УДАЛЕННОГО СЕРВЕРА (с 40 сотрудниками):

**Что делает:** Экспортирует данные входа/выхода из локальной базы на Google сервер каждый день в 7:00.

```powershell
# PowerShell от имени администратора на удаленном сервере
cd export_to_database
.\setup_remote_worktime_export.ps1
```

**Данные будут отображаться в:** Меню → Мониторинг времени → **Отчет Удаленка**

📖 Подробная инструкция: `ШПАРГАЛКА_ЭКСПОРТ_УДАЛЕНКА.md`

---

### 💻 ДЛЯ ЛОКАЛЬНЫХ ПК (индивидуальный мониторинг):

#### Для новых ПК (первая установка):
```powershell
# PowerShell от имени администратора
.\setup_pc_monitoring.ps1
```

#### Для ПК со старой версией (переустановка):
```powershell
# PowerShell от имени администратора
.\ПЕРЕУСТАНОВИТЬ.ps1
```

## 📁 Структура файлов

### 🏢 Скрипты для УДАЛЕННОГО СЕРВЕРА (где работают 40 человек):

| Файл | Назначение |
|------|-----------|
| `export_remote_worktime.ps1` | **Экспорт данных с удаленного сервера** - читает из базы messenger.db и отправляет на Google сервер |
| `setup_remote_worktime_export.ps1` | **Установщик экспорта** - создает задачу планировщика для автоматической отправки в 7:00 |
| `Mesendger_Remote_Worktime_Export.xml` | **XML конфигурация** - задача для планировщика (ежедневно в 7:00) |

📖 **Документация для удаленного сервера:**
- `УСТАНОВКА_ЭКСПОРТА_НА_УДАЛЕННЫЙ_СЕРВЕР.md` - полная инструкция по установке
- `ШПАРГАЛКА_ЭКСПОРТ_УДАЛЕНКА.md` - быстрая установка за 5 минут
- `АНАЛИЗ_ПРОБЛЕМЫ_ЭКСПОРТА.md` - решение проблемы "данные приходят не всегда"

---

### 💻 Скрипты для ЛОКАЛЬНЫХ ПК (индивидуальный мониторинг):

| Файл | Назначение |
|------|-----------|
| `send_pc_startup.ps1` | Отправка данных о **включении** ПК + данные о выключении из журнала Windows |
| `send_activity.ps1` | Отправка **скриншотов**, данных о посещенных **сайтах** и открытых **приложениях** |

### ⚙️ Установщики и утилиты:

| Файл | Назначение |
|------|-----------|
| `setup_pc_monitoring.ps1` | **Основной установщик** - настройка мониторинга с нуля |
| `ПЕРЕУСТАНОВИТЬ.ps1` | **Переустановка** - удаляет старые задачи и устанавливает новые (сохраняет username) |
| `УДАЛИТЬ_СТАРЫЕ_ЗАДАЧИ.ps1` | **Удаление** - только удаляет задачи и файлы (без установки) |

### 📄 XML файлы задач планировщика:

| Файл | Триггер | Назначение |
|------|---------|-----------|
| `Mesendger_PC_Startup_Monitor.xml` | При входе пользователя (задержка 30 сек) | Запускает `send_pc_startup.ps1` |
| `Mesendger_PC_Shutdown_Monitor.xml` | EventID 1074 (выключение системы) | Запускает `send_pc_shutdown_with_fallback.ps1` |
| `Mesendger_PC_Activity_Monitor.xml` | При входе пользователя | Запускает `send_activity.ps1` в режиме loop |
| `Mesendger_PC_Logoff_Monitor.xml` | SessionDisconnect/Lock | Запускает `send_pc_shutdown_with_fallback.ps1` |
| `Mesendger_PC_Shutdown_Monitor_Background.xml` | При входе пользователя | Запускает `monitor_shutdown.ps1` в фоне |

### 📚 Инструкции:

| Файл | Содержание |
|------|-----------|
| `УСТАНОВКА_НА_15_ПК.md` | **Подробная инструкция** по установке на 15 компьютеров |
| `ШПАРГАЛКА_УСТАНОВКА.md` | **Краткая шпаргалка** для быстрой установки |
| `ШПАРГАЛКА_ПЕРЕУСТАНОВКА.md` | **Краткая шпаргалка** для переустановки на ПК со старыми версиями |
| `ФИНАЛЬНОЕ_ИСПРАВЛЕНИЕ_ВРЕМЕНИ.md` | Техническая документация по исправлению времени выключения |
| `Инструкция доступа Энидеск.txt` | Инструкция по удаленному доступу через AnyDesk |

## 📊 Что устанавливается на каждый ПК:

### Файлы на ПК:
```
C:\pc-worktime\
├── send_pc_startup.ps1
├── send_pc_shutdown_with_fallback.ps1
├── send_activity.ps1
├── monitor_shutdown.ps1
└── XML файлы задач (5 файлов)

%APPDATA%\mesendger\
├── agent_config.json (username и настройки)
├── pc_startup.log (лог включений)
├── pc_shutdown.log (лог выключений)
├── activity.log (лог активности)
└── screenshots\ (временная папка для скриншотов)
```

### Задачи в планировщике:
- ✅ **Mesendger_PC_Startup_Monitor** - отправка включения
- ✅ **Mesendger_PC_Shutdown_Monitor** - отправка выключения
- ✅ **Mesendger_PC_Activity_Monitor** - скриншоты, сайты, приложения
- ⚠️ **Mesendger_PC_Logoff_Monitor** - дополнительный триггер выхода (опционально)
- ⚠️ **Mesendger_PC_Shutdown_Monitor_Background** - фоновый мониторинг (опционально)

## ⚙️ Настройки

### Сервер:
- **URL:** `http://35.232.108.72`
- **API:** `/api/worktime` (включение/выключение)
- **API:** `/api/activity-log-batch` (сайты/приложения)
- **API:** `/api/activity-screenshot` (скриншоты)

### Интервалы по умолчанию:
- **Скриншоты:** каждые 5 минут
- **Сайты:** каждые 5 минут
- **Приложения:** каждые 5 минут

Интервалы можно изменить при установке или в файле:
```
%APPDATA%\mesendger\agent_config.json
```

## 🔍 Проверка работы

### Проверить задачи:
```powershell
schtasks /Query /TN Mesendger_PC_Startup_Monitor
schtasks /Query /TN Mesendger_PC_Shutdown_Monitor
schtasks /Query /TN Mesendger_PC_Activity_Monitor
```

### Проверить логи:
```powershell
notepad $env:APPDATA\mesendger\pc_startup.log
notepad $env:APPDATA\mesendger\pc_shutdown.log
notepad $env:APPDATA\mesendger\activity.log
```

### Проверить файлы:
```powershell
dir C:\pc-worktime
dir $env:APPDATA\mesendger
```

## 🐛 Устранение проблем

### Скрипты не запускаются:
```powershell
Set-ExecutionPolicy RemoteSigned -Scope CurrentUser
```

### Нет доступа к серверу:
```powershell
Test-NetConnection 35.232.108.72 -Port 80
```

### Неправильный username:
```powershell
notepad $env:APPDATA\mesendger\agent_config.json
# Исправьте username и перезапустите задачи
```

### Задачи не создаются:
- Убедитесь что PowerShell запущен **от имени администратора**
- Проверьте что XML файлы находятся в той же папке

## 📞 Техническая информация

### Системные требования:
- Windows 10/11
- PowerShell 5.1 или выше
- Права администратора (для установки)
- Доступ к серверу `http://35.232.108.72`

### Совместимость:
- ✅ Windows 10
- ✅ Windows 11
- ✅ Windows Server 2016+
- ⚠️ Требуется интерактивный вход пользователя (для скриншотов)

### Безопасность:
- Скрипты работают только от имени конкретного пользователя
- Данные отправляются по HTTP (не зашифрованы)
- Логи сохраняются локально в `%APPDATA%\mesendger\`

## 🔄 История версий

### Версия 3.0 (текущая) - 1 декабря 2025
- ✅ Исправлено время выключения (читается из журнала Windows при включении)

### Ключевые особенности:
- **Точное время выключения:** читается из журнала Windows (EventID 6006)
- **Fallback механизм:** если не удалось отправить сразу - сохраняется в файл
- **Автоматическая отправка:** при следующем включении отправляются несохраненные данные
