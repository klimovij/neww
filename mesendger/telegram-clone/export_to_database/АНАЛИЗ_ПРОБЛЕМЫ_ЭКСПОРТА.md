## 🐛 ВОЗМОЖНЫЕ ПРИЧИНЫ И ИСПРАВЛЕНИЯ

### 1. ❌ ПРОБЛЕМА: Неправильный SQL запрос

**Было в оригинальном коде:**
```sql
WHERE (
  (substr(event_time,5,1)='-' AND substr(event_time,1,10) = ?) 
  OR 
  (substr(event_time,7,4)||'-'||...) = ?)
)
```

**Проблема:** 
- Не проверялась длина строки `event_time` перед `substr()`
- Если `event_time` короче 10 символов → **ошибка или пропуск**
- Формат даты мог быть разным в разных записях

**✅ ИСПРАВЛЕНО:**
```sql
WHERE (
  (length(event_time) >= 10 AND substr(event_time,5,1)='-' ...) 
  OR 
  (length(event_time) >= 10 AND substr(event_time,3,1)='.' ...)
  OR 
  (date(event_time) = ?)
)
```

**Что улучшилось:**
- ✅ Проверка длины строки перед `substr()`
- ✅ Дополнительная проверка через `date(event_time)`
- ✅ Поддержка **всех** форматов дат SQLite

### 3. ❌ ПРОБЛЕМА: Неправильная обработка даты

**Было:**
```javascript
const parts = targetDate.split('-');
const ddmmyyyy = parts[2] + '.' + parts[1] + '.' + parts[0];
```

**Проблема:**
- Если формат даты не `YYYY-MM-DD` → **массив `parts` неправильный**
- Переменная `ddmmyyyy` может быть неверной

**✅ ИСПРАВЛЕНО:**
```javascript
const parts = targetDate.split('-');
const ddmmyyyy = parts.length === 3 
  ? (parts[2] + '.' + parts[1] + '.' + parts[0]) 
  : '';
```

**Что улучшилось:**
- ✅ Проверка `parts.length`
- ✅ Безопасная конвертация формата

### 4. ❌ ПРОБЛЕМА: event_id может отсутствовать

**Было:**
```javascript
event_id: row.event_id
```

**Проблема:**
- Если `event_id` не заполнен в базе → отправляется `null` или `undefined`
- Сервер может отклонить запись

**✅ ИСПРАВЛЕНО:**
```javascript
event_id: row.event_id || (row.event_type === 'logout' ? 4634 : 4624)
```

**Что улучшилось:**
- ✅ Автоматическое определение `event_id` по типу события
- ✅ Всегда корректное значение

### 5. ❌ ПРОБЛЕМА: Отсутствие даты по умолчанию

**Было:**
```powershell
param(
    [Parameter(Mandatory = $true)]
    [string]$Date
)
```

**Проблема:**
- При автоматическом запуске в 7:00 **дата НЕ передавалась**
- Скрипт **падал с ошибкой** "Parameter Date is required"

**✅ ИСПРАВЛЕНО:**
```powershell
param(
    [Parameter(Mandatory = $false)]
    [string]$Date
)

if (-not $Date) {
    $yesterday = (Get-Date).AddDays(-1)
    $Date = $yesterday.ToString("yyyy-MM-dd")
}
```

**Что улучшилось:**
- ✅ Автоматически берет **вчерашний день**
- ✅ Идеально для планировщика (запуск в 7:00 → отправляет данные за вчера)
- ✅ Можно указать дату вручную

### 6. ❌ ПРОБЛЕМА: Недостаточная проверка путей

**Было:**
```powershell
if (-not (Test-Path $dbPath)) {
    Write-Host "Database file not found"
    exit 1
}
```

**Проблема:**
- Не показываются **ожидаемые пути**
- Непонятно, где искать базу

**✅ ИСПРАВЛЕНО:**
```powershell
if (-not (Test-Path $dbPath)) {
    Write-Host "Database file not found: $dbPath" -ForegroundColor Red
    Write-Host "Expected paths:" -ForegroundColor Yellow
    Write-Host "  1. $(Join-Path $scriptRoot '..\server\messenger.db')"
    Write-Host "  2. $(Join-Path $scriptRoot '..\..\server\messenger.db')"
    exit 1
}
Write-Host "✅ Found database: $dbPath" -ForegroundColor Green
```

**Что улучшилось:**
- ✅ Показываются **все** ожидаемые пути
- ✅ Подтверждение успешного нахождения базы
- ✅ Цветной вывод для быстрого поиска проблем

### 7. ❌ ПРОБЛЕМА: Нет проверки sqlite3 модуля

**Было:**
```powershell
# Никакой проверки
```

**Проблема:**
- Если модуль `sqlite3` не установлен → **скрипт падает**
- Ошибка `Cannot find module 'sqlite3'`

**✅ ИСПРАВЛЕНО:**
```powershell
$sqlite3Path = Join-Path $serverDir "node_modules\sqlite3"
if (-not (Test-Path $sqlite3Path)) {
    Write-Host "❌ sqlite3 module not found"
    Write-Host "Run: cd $serverDir && npm install sqlite3"
    exit 1
}
Write-Host "✅ sqlite3 module found" -ForegroundColor Green
```

**Что улучшилось:**
- ✅ Проверка наличия модуля
- ✅ Инструкция по установке
- ✅ Понятное сообщение об ошибке

### 8. ❌ ПРОБЛЕМА: Таймаут при отправке

**Было:**
```powershell
Invoke-RestMethod -Uri $apiUrl -Method POST ...
```

**Проблема:**
- Если сервер медленный или данных много → **таймаут**
- По умолчанию таймаут = **100 секунд**, но сеть может быть медленной

**✅ ИСПРАВЛЕНО:**
```powershell
Invoke-RestMethod -Uri $apiUrl -Method POST ... -TimeoutSec 30
```

**Что улучшилось:**
- ✅ Явный таймаут 30 секунд
- ✅ Быстрый фейл при проблемах с сетью
- ✅ Понятная ошибка вместо зависания

### 9. ❌ ПРОБЛЕМА: XML задача неправильно настроена

**Было в XML:**
```xml
<Command>powershell.exe</Command>
<Arguments>-File "путь\export_remote_worktime.ps1"</Arguments>
```

**Проблема:**
- ExecutionPolicy может блокировать выполнение
- Окно PowerShell может появляться и отвлекать

**✅ ИСПРАВЛЕНО:**
```xml
<Command>powershell.exe</Command>
<Arguments>-NoProfile -ExecutionPolicy Bypass -WindowStyle Hidden -File "..."</Arguments>
```

**Что улучшилось:**
- ✅ `-NoProfile` → быстрый запуск
- ✅ `-ExecutionPolicy Bypass` → обход политик
- ✅ `-WindowStyle Hidden` → скрытое выполнение
